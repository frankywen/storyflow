## 一、配音/字幕系统

### 1.1 配音生成流程

```
场景对话提取 → 角色音色匹配 → TTS生成音频 → 音频预览 → 视频合成
```

**流程说明：**

1. 从故事解析结果提取每个场景的角色对话和旁白内容
2. 根据角色属性（性别/年龄）自动匹配音色，用户可手动调整
3. 调用TTS服务（云端API或Edge-TTS）生成音频文件
4. 用户可预览每个音频，不满意可重新生成或调整音色
5. 确认后将音频与视频合成，生成带配音和字幕的最终视频

**配音内容结构示例：**
```json
{
  "scene_id": 1,
  "dialogues": [
    {"character": "小明", "text": "今天天气真好！"},
    {"character": "小红", "text": "是啊，我们去公园吧。"}
  ],
  "narration": "小明和小红走在阳光明媚的街道上"
}
```

**边界情况处理：**

| 情况 | 处理方式 |
|------|----------|
| 场景无对话 | 只生成旁白音频（如果有） |
| 场景无旁白 | 只生成对话音频 |
| 场景无对话也无旁白 | 跳过该场景的配音生成 |
| 角色未分配音色 | 使用默认中性声音 |
| 角色性别/年龄未知 | 使用默认中性声音 |

**TTS服务降级方案：**

当配置的TTS Provider不可用时，按以下顺序降级：

```
1. 尝试配置的主Provider
   ↓ 失败
2. 尝试备用Provider（需配置）
   ↓ 失败
3. 返回错误，提示用户稍后重试
```

**配置示例：**
```env
TTS_PROVIDER=edge-tts
TTS_FALLBACK_PROVIDER=volcengine  # 备用Provider
```

**音频文件存储方案：**

| 存储方式 | 配置 | 说明 |
|----------|------|------|
| 本地文件系统 | `STORAGE_TYPE=local` | 存储在服务器本地，适合单机部署 |
| 阿里云OSS | `STORAGE_TYPE=oss` | 云存储，适合生产环境 |
| 腾讯云COS | `STORAGE_TYPE=cos` | 云存储，适合生产环境 |

**本地存储配置：**
```env
STORAGE_TYPE=local
STORAGE_LOCAL_PATH=./uploads/audio
AUDIO_BASE_URL=http://localhost:8080/uploads/audio
```

### 1.2 TTS服务接口设计

**统一接口：**
```go
type TTSProvider interface {
    GenerateVoice(text string, voiceID string) (*AudioResult, error)
    GetAvailableVoices() ([]Voice, error)
}

type AudioResult struct {
    AudioURL    string  // 生成的音频URL
    Duration    float64 // 音频时长（秒）
    VoiceID     string  // 使用的音色ID
}

type TTSConfig struct {
    Provider    string
    BaseURL     string
    APIKey      string
    AppID       string  // 部分服务需要
}
```

**支持的Provider：**

| Provider | 配置值 | 特点 |
|----------|--------|------|
| Edge-TTS | `edge-tts` | Microsoft免费服务、无需API Key、通过命令行调用 |
| 火山引擎 | `volcengine` | 国内稳定、按量计费、音色丰富 |
| 阿里云 | `alibaba` | 国内稳定、按量计费 |

**Edge-TTS说明：**
- Edge-TTS是Microsoft提供的免费TTS服务，通过Python库`edge-tts`调用
- 无需API Key，无需本地服务部署
- 安装方式：`pip install edge-tts`
- 调用方式：直接在Go中通过`exec.Command`调用edge-tts命令行工具
- **音频时长获取**：Edge-TTS生成音频后，使用ffprobe获取音频时长
  ```bash
  # 生成音频
  edge-tts --text "你好" --voice zh-CN-XiaoxiaoNeural --write-media output.mp3

  # 获取时长
  ffprobe -i output.mp3 -show_entries format=duration -v quiet -of csv="p=0"
  ```

**配置示例：**
```env
# TTS Provider选择
TTS_PROVIDER=edge-tts  # edge-tts / volcengine / alibaba

# Edge-TTS命令路径（默认edge-tts，如已加入PATH可省略）
TTS_EDGE_CMD=edge-tts

# 火山引擎
TTS_VOLCENGINE_URL=https://openspeech.bytedance.com/api/v1/tts
TTS_VOLCENGINE_API_KEY=your-api-key
TTS_VOLCENGINE_APP_ID=your-app-id

# 阿里云
TTS_ALIBABA_URL=https://nls-gateway.cn-shanghai.aliyuncs.com/stream/v1/tts
TTS_ALIBABA_API_KEY=your-api-key
TTS_ALIBABA_APP_KEY=your-app-key
```

### 1.4 字幕生成系统

**字幕生成流程：**
```
配音音频 → 提取时间轴 → 生成字幕文件(SRT/VTT) → 视频合成时嵌入
```

**音频时间轴计算逻辑：**

字幕的start_time和end_time根据音频文件顺序累加计算：

```
场景1:
  - 对话1: 0.0s - 2.5s (duration=2.5s)
  - 对话2: 2.5s - 5.0s (duration=2.5s)
  - 旁白: 5.0s - 8.0s (duration=3.0s)
场景2:
  - 对话1: 8.0s - 10.0s (累加场景1总时长后开始)
  - ...
```

**计算代码示例：**
```go
func CalculateTimeAxis(audioFiles []AudioFile) []Subtitle {
    var subtitles []Subtitle
    currentTime := 0.0

    for _, audio := range audioFiles {
        subtitles = append(subtitles, Subtitle{
            Text:      audio.TextContent,
            StartTime: currentTime,
            EndTime:   currentTime + audio.Duration,
        })
        currentTime += audio.Duration
    }

    return subtitles
}
```

**字幕数据表：**
```sql
CREATE TABLE subtitles (
    id SERIAL PRIMARY KEY,
    story_id INT REFERENCES stories(id) ON DELETE CASCADE,
    scene_id INT REFERENCES scenes(id) ON DELETE CASCADE,
    subtitle_type VARCHAR(20) NOT NULL,    -- dialogue / narration
    character_id INT REFERENCES characters(id) ON DELETE SET NULL,
    text TEXT NOT NULL,
    start_time FLOAT NOT NULL,             -- 开始时间（秒）
    end_time FLOAT NOT NULL,               -- 结束时间（秒）
    style_config JSONB,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_subtitles_story ON subtitles(story_id);
CREATE INDEX idx_subtitles_scene ON subtitles(scene_id);
```

**字幕样式配置：**

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| position | bottom-center | 底部居中 |
| font_size | 24 | 字体大小(px) |
| font_color | #FFFFFF | 白色文字 |
| background_color | #00000080 | 半透明黑背景 |
| font_family | Arial | 字体 |

**style_config JSON结构：**
```json
{
  "position": "bottom-center",
  "font_size": 24,
  "font_color": "#FFFFFF",
  "background_color": "#00000080",
  "font_family": "Arial"
}
```

**字幕显示模式：**

| 模式 | 配置值 | 说明 |

| narration_only | `narration_only` | 只显示旁白 |

### 1.6 音频与视频合成

**合成流程：**
```
无配音视频 → 导入配音音频 → 导入字幕 → ffmpeg合成 → 带配音字幕视频
```

**合成任务表：**
```sql
CREATE TABLE audio_video_synthesis (
    id SERIAL PRIMARY KEY,
    story_id INT REFERENCES stories(id) ON DELETE CASCADE,
    video_url VARCHAR(500),                -- 无配音视频URL
    audio_urls JSONB,                      -- 各场景音频URL列表
    subtitle_url VARCHAR(500),             -- 字幕文件URL
    output_url VARCHAR(500),               -- 合成后视频URL
    status VARCHAR(20) DEFAULT 'pending',  -- pending / processing / completed / failed
    progress INT DEFAULT 0,                -- 合成进度 0-100
    error_message TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    completed_at TIMESTAMP
);

CREATE INDEX idx_synthesis_story ON audio_video_synthesis(story_id);
```

**audio_urls JSONB结构：**
```json
[
  {"scene_id": 1, "audio_url": "https://.../scene1.mp3", "duration": 8.5},
  {"scene_id": 2, "audio_url": "https://.../scene2.mp3", "duration": 6.2},
  {"scene_id": 3, "audio_url": "https://.../scene3.mp3", "duration": 10.1}
]
```

**ffmpeg合成命令：**

多场景音频合并需要先将所有音频拼接成一个完整音轨：

```bash
# 步骤1: 创建音频文件列表
echo "file 'scene1.mp3'" > audio_list.txt
echo "file 'scene2.mp3'" >> audio_list.txt
echo "file 'scene3.mp3'" >> audio_list.txt

# 步骤2: 拼接音频文件
ffmpeg -f concat -safe 0 -i audio_list.txt -c copy audio_track.mp3

# 步骤3: 合成视频+音频+字幕
ffmpeg -i input_video.mp4 \
       -i audio_track.mp3 \
       -vf "subtitles=subtitles.srt:force_style='Fontsize=24,PrimaryColour=&HFFFFFF'" \
       -c:v libx264 -c:a aac \
       -shortest \                          # 以最短的流为准
       output_with_audio_subs.mp4
```

**视频/音频时长不匹配处理：**

| 情况 | 处理方案 |
|------|----------|
| 音频短于视频 | 视频保持原样，音频结束后静音 |
| 音频长于视频 | 使用`-shortest`参数，截断音频到视频时长 |
| 场景音频需与场景视频同步 | 按场景顺序排列音频，每个场景视频独立合成后合并 |

**场景级合成流程（推荐）：**
```
场景1视频 + 场景1音频 → 场景1合成视频
场景2视频 + 场景2音频 → 场景2合成视频
...
所有场景视频 → 最终视频合并
```

**长度检查代码：**
```go
func ValidateTextForTTS(text string) error {
    if len(text) == 0 {
        return errors.New("文本不能为空")
    }
    if len(text) > 500 {
        return errors.New("单条文本最长500字符，请分段处理")
    }
    return nil
}

// 长文本自动分段
func SplitLongText(text string, maxLength int) []string {
    if len(text) <= maxLength {
        return []string{text}
    }
    // 按句子分割
    sentences := regexp.MustCompile(`[。！？.!?]`).Split(text, -1)
    var result []string
    var current string
    for _, s := range sentences {
        if len(current)+len(s) > maxLength {
            result = append(result, current)
            current = s
        } else {
            current += s
        }
    }
    if current != "" {
        result = append(result, current)
    }
    return result
}
```

**批量配音生成异步处理：**

配音生成采用异步任务处理，流程如下：

```
POST /api/v1/audio/generate
      ↓
创建任务记录，返回task_id
      ↓
后台goroutine逐场景生成音频
      ↓
GET /api/v1/audio/status/:task_id 查询进度
```

**任务状态表设计：**
```sql
CREATE TABLE audio_generation_tasks (
    id SERIAL PRIMARY KEY,
    story_id INT REFERENCES stories(id) ON DELETE CASCADE,
    status VARCHAR(20) DEFAULT 'pending',  -- pending / processing / completed / failed
    total_scenes INT DEFAULT 0,            -- 总场景数
    completed_scenes INT DEFAULT 0,        -- 已完成场景数
    progress INT DEFAULT 0,                -- 进度百分比 0-100
    error_message TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    completed_at TIMESTAMP
);
```

**并发控制：**
- 使用goroutine池控制并发数（默认3个并发）
- 每个音频生成完成后更新进度
- 失败重试机制：最多重试3次

**并发任务数限制：**

| 限制项 | 默认值 | 配置环境变量 | 说明 |
|--------|--------|--------------|------|
| 同时进行的配音任务数 | 5 | `MAX_AUDIO_TASKS` | 超过后返回"系统繁忙" |
| 单任务并发TTS请求数 | 3 | `MAX_TTS_CONCURRENT` | 每个任务内的并发数 |
| 单场景最大音频数 | 20 | `MAX_AUDIOS_PER_SCENE` | 对话+旁白总数限制 |

**限流逻辑：**
```go
var activeAudioTasks int32

func StartAudioGeneration(storyID int) error {
    // 检查并发任务数
    if atomic.LoadInt32(&activeAudioTasks) >= MaxAudioTasks {
        return errors.New("系统繁忙，请稍后重试")
    }

    atomic.AddInt32(&activeAudioTasks, 1)
    defer atomic.AddInt32(&activeAudioTasks, -1)

    // 执行生成任务...
}
```

**部分场景配音失败处理策略：**

| 策略 | 说明 |

**预览生成逻辑：**

func Logout(token string) error {
    claims := ParseJWT(token)
    // 将Token加入黑名单
    db.Create(&TokenBlacklist{
        TokenJti:  claims.Jti,
        UserID:    claims.UserID,
        ExpiresAt: time.Unix(claims.Exp, 0),

}

// Token验证时检查黑名单
func ValidateToken(token string) error {

    return nil
}

|----------|------|

**验证失败次数存储方案：**

使用Redis存储验证失败次数，支持快速过期：

```go
// Redis Key设计
// verification:failed:{email} -> 失败次数
// verification:locked:{email} -> 锁定标记

const (
    MaxFailedAttempts = 5
    LockDuration      = 15 * time.Minute
)

func IncrementFailedAttempts(email string) error {
    key := fmt.Sprintf("verification:failed:%s", email)
    count, _ := redis.Incr(key).Result()
    redis.Expire(key, LockDuration)

    if count >= MaxFailedAttempts {
        // 锁定邮箱
        lockKey := fmt.Sprintf("verification:locked:%s", email)
        redis.Set(lockKey, "1", LockDuration)
    }
    return nil
}

func IsLocked(email string) bool {
    lockKey := fmt.Sprintf("verification:locked:%s", email)
    return redis.Exists(lockKey).Val() > 0
}

func ClearFailedAttempts(email string) {
    key := fmt.Sprintf("verification:failed:%s", email)
    redis.Del(key)
}
```

**验证码验证逻辑：**
```go
func VerifyCode(email, code string) error {
    // 1. 检查是否被锁定
    if IsLocked(email) {
        return errors.New("验证次数过多，请15分钟后再试")
    }

    // 2. 查询验证码
    record := GetValidCode(email, code)
    if record == nil {
        IncrementFailedAttempts(email)  // 增加失败次数
        return errors.New("验证码错误")
    }

    // 3. 检查是否过期
    if time.Now().After(record.ExpiresAt) {
        return errors.New("验证码已过期")
    }

    // 4. 标记已使用
    MarkCodeAsUsed(record.ID)

    // 5. 清除失败记录
    ClearFailedAttempts(email)

    return nil
}
```

  user: User | null;
  isLoading: boolean;
  isAuthenticated: boolean;
  login: (email: string, password: string) => Promise<void>;
  logout: () => Promise<void>;
  refreshToken: () => Promise<void>;

export const AuthProvider: React.FC<{ children: React.ReactNode }> = ({ children }) => {
  const [user, setUser] = useState<User | null>(null);
  const [isLoading, setIsLoading] = useState(true);

  // 初始化时检查Token有效性
  useEffect(() => {
    checkAuth();
  }, []);

  const checkAuth = async () => {
    try {
      const token = localStorage.getItem('token');
      if (token) {
        const userData = await api.get('/api/v1/auth/me');
        setUser(userData);
      }
    } catch {
      localStorage.removeItem('token');
    } finally {
      setIsLoading(false);
    }
  };

  return (
    <AuthContext.Provider value={{ user, isLoading, isAuthenticated: !!user, login, logout, refreshToken }}>
      {children}
    </AuthContext.Provider>
  );
};
